# ValYoued

AI-powered multi-asset valuation app. Users pick an asset category (luxury watches, designer bags, classic cars, fine art, sneakers, trading cards, wine, jewelry, small businesses), fill in a dynamic form, and receive a valuer-grade report with baseline range, market sentiment overlay, international arbitrage table, and (Pro tier) negotiation tactics.

## Architecture

Monorepo (pnpm workspaces).

- `lib/api-spec/openapi.yaml` ŌĆö single source of truth for the API
- `lib/api-client-react`, `lib/api-zod` ŌĆö generated by orval from the spec
- `lib/db` ŌĆö drizzle schema; tables: `valuations` (legacy, unused), `estimates`, `listings`, `platform_events`, `billing_subscriptions`
- `lib/llm` ŌĆö vendor-agnostic LLM surface (`getLlm`, `defaultModel`): Anthropic or OpenAI behind env (`LLM_PROVIDER`, or vendor keys; optional neutral `LLM_API_KEY` + `LLM_BASE_URL`)

### Artifacts

- `artifacts/api-server` (Express) ŌĆö endpoints under `/api/*`:
  - `/api/asset-types`, `/api/regions`, `/api/estimates` (POST/GET/list/stats), `/api/healthz`
  - `/api/me/billing`, `/api/me/data-export` ŌĆö subscription snapshot + GDPR JSON bundle (authenticated)
  - `/api/admin/overview`, `/api/admin/users/:userId/activity` ŌĆö ops dashboard (`ADMIN_USER_IDS` env: comma-separated Clerk user IDs)
  - `/api/billing/checkout-session`, `/api/billing/customer-portal`, `/api/billing/webhook` ŌĆö Stripe (`STRIPE_SECRET_KEY`, `STRIPE_PRICE_PRO`, `STRIPE_WEBHOOK_SECRET`, `PUBLIC_APP_URL`). Webhook mounted before `express.json()` with raw body.
  - `/api/vision/extract` ŌĆö Anthropic vision (`src/lib/vision.ts`). Rate-limited to 8 req/min/IP.
  - `/api/listings` ŌĆö marketplace drafts via Anthropic (`src/lib/listing.ts`). Rate-limited to 12 req/min/IP.
  - `src/lib/assetTypes.ts` ŌĆö catalog of asset categories and dynamic form fields (20+ asset classes)
  - `src/lib/estimate.ts` ŌĆö Anthropic claude-sonnet-4-5 prompt ŌåÆ JSON; net market factor + arbitrage; heuristic fallback
  - `src/lib/platformEvents.ts` ŌĆö append-only `platform_events` on estimate / listing / vision
  - `src/lib/rateLimit.ts` ŌĆö in-memory IP buckets on AI routes
  - JSON body limit: 8MB (allows ~5MB photos base64-encoded)
- `artifacts/valuation-app` (React + Vite + wouter + shadcn) ŌĆö deep-ink/brushed-gold "private banker terminal" UI
  - Pages: `/`, `/estimate/new`, `/estimates`, `/estimates/:id`, `/portfolio`, `/listings`, `/stats`, `/settings`, `/admin`, public `/privacy`, `/about`
  - `/portfolio` ŌĆö dashboard with USD-equivalent total (FX_TO_USD constants), recharts pie chart + Herfindahl diversification score, three tabs (Albums / Asset boxes / Table). Asset boxes have a "List for sale" button that opens the ad-generator dialog.
  - `/listings` ŌĆö saved ad drafts grid (platform badge, suggested price, snippet) ŌåÆ click for full draft view dialog with copy-to-clipboard, photo angles, hashtags, pro tips. Delete with confirm.
  - `PhotoUploadCard` (used on `/estimate/new`) ŌĆö drag-drop photo ŌåÆ AI extracts attributes ŌåÆ calls `form.setValue` for all matched fields. Skips year of purchase, mileage, original price (user-only data). Uses request tokens to ignore stale responses.
  - `GenerateListingDialog` ŌĆö platform select (auto-suggested from asset class) + pricing strategy + AI generation. Mounted both on `/portfolio` asset boxes and `/estimates/:id` report header.
  - Pro tier toggle persisted to `localStorage["valyoued.pro"]`; passed to `useCreateEstimate` as `params.pro`. **Server-backed Pro** uses Stripe + `billing_subscriptions.tier` (`GET /api/me/billing`) ŌĆö wire UI to prefer subscription tier when you deprecate the local toggle.

### Mobile (parity shell)

- `mobile/` ŌĆö standalone Expo mini-app (not in the pnpm workspace; avoids React-version coupling). Full-feature parity by loading the deployed web app inside `react-native-webview`. Set `EXPO_PUBLIC_VALYOUED_WEB_URL` to your hosted `valuation-app`. Native Clerk + shared cookies can replace the WebView in a later phase.

### Payments (two economical options)

1. **Stripe Checkout + Customer Portal** (implemented) ŌĆö lowest integration surface for subscriptions; you remain merchant of record for card disputes.
2. **Paddle or Lemon Squeezy** (documented in Settings UI) ŌĆö merchant-of-record model simplifies EU/UK VAT; map their subscription webhooks into `billing_subscriptions` like Stripe.

### Deploying the web UI to Vercel

1. Import this repo in Vercel ŌĆö leave **Root Directory** at `.` so `pnpm-workspace.yaml` resolves.
2. Vercel reads [`vercel.mjs`](vercel.mjs): installs with **pnpm**, builds `@workspace/valuation-app`, serves `artifacts/valuation-app/dist/public`, and rewrites **`/api/*`** to **`VITE_API_ORIGIN`** when that env var is set (so JSON routes are not served as the SPA).
3. Set **`VITE_CLERK_PUBLISHABLE_KEY`** (and optional **`VITE_CLERK_PROXY_URL`**) in the Vercel project env vars.
4. Host the Express API on a Node-friendly platform (Railway, Fly.io, Render, ECS, etc.). Set **`VITE_API_ORIGIN`** on Vercel to that APIŌĆÖs public origin (no trailing slash). The UI sends Clerk **`Authorization: Bearer`** tokens automatically for cross-origin calls.
5. In Clerk, add your Vercel domain under **Allowed origins / redirect URLs**. Point Stripe **`PUBLIC_APP_URL`** at the Vercel site URL.

**Why API + DB usually stay off Vercel:** photo valuation payloads can exceed Vercel serverless body limits; long-lived Postgres + Stripe raw webhooks map cleanly to a single Node service.

## Proprietary model data path

Valuation outputs and user flows append structured rows to `platform_events` (`estimate.created`, `listing.created`, `vision.extracted`). Combine with `estimates.result` JSON (full AI report) for offline extract / training pipelines. Add explicit consent flags and anonymisation jobs before large-scale model training.

## Conventions

- API: orval generates `Get*Response`/`Create*Body` zod schemas (api-zod) and `use*` hooks (api-client-react). Re-run codegen with `pnpm --filter @workspace/api-spec run codegen` after editing `openapi.yaml`.
- DB push: `pnpm --filter @workspace/db run push-force`
- Frontend imports hooks ONLY from `@workspace/api-client-react`.
- LLM calls: `import { getLlm, defaultModel } from "@workspace/llm"`. Configure with `LLM_PROVIDER` and `LLM_API_KEY`, or vendor keys (`OPENAI_API_KEY` / `ANTHROPIC_API_KEY`); optional `LLM_BASE_URL`. Defaults: Anthropic `claude-sonnet-4-5`, OpenAI `gpt-4o`; override with `LLM_MODEL`.

## Authentication & Onboarding

- **Auth provider**: Replit-managed Clerk (`@clerk/express` on api-server, `@clerk/react` on the web app). `setupClerkWhitelabelAuth` provisioned the app; `CLERK_*` secrets + `VITE_CLERK_PUBLISHABLE_KEY` are set automatically.
- **API**: `app.ts` mounts the Clerk proxy + `clerkMiddleware` before body parsers. `requireAuth` middleware (in `src/middlewares/requireAuth.ts`) gates all writes (POST/DELETE on `/api/estimates`, `/api/listings`, `/api/vision/extract`) and the per-estimate read (`GET /api/estimates/:id`). List/stats GETs are scoped by `userId`. The `userId` columns on `estimatesTable` and `listingsTable` are nullable so legacy rows survive but are invisible to all users.
- **Web routing** (`App.tsx`): `/`, `/sign-in`, `/sign-up`, `/start`, `/about`, `/privacy` are full-bleed (no sidebar, no auth gate). All other routes (`/dashboard`, `/estimate/new`, `/estimates`, `/portfolio`, `/listings`, `/settings`, `/admin`, ŌĆ”) are wrapped in `AuthGate` which redirects signed-out users to `/sign-in`. `Show when="signed-in|signed-out"` (Clerk's component) drives the gating; `routerPush/routerReplace` are wired into wouter so Clerk navigation respects the artifact base path.
- **Landing page** (`pages/landing.tsx`): signed-out `/` shows a futuristic dark hero with a 3D globe (`components/Globe.tsx` ŌåÆ react-globe.gl + three) auto-rotating with 15 worldwide deal markers + arcs to buyer locations. The globe is wrapped in an error boundary that falls back to a stylized circle when WebGL is unavailable (e.g. headless screenshot environments).
- **Onboarding gate** (`pages/start.tsx` + `pages/estimates/new.tsx`): anonymous users can fill the valuation form on `/start`. On submit while signed-out, the payload is stashed to `sessionStorage["valyoued.pendingEstimate"]` and the user is redirected to `/sign-up?redirect_url=/start`. After Clerk completes sign-up and redirects back, an effect on mount auto-submits the pending payload (using a `resumedRef` to prevent double-submit) and routes to `/estimates/:id`.
- **Marketplace post-on**: `lib/platforms.ts` exports a `PLATFORM_URL` map of "post a listing" URLs for each marketplace. `ListingDraftView` shows a prominent "Post on {Platform} ŌåÆ" external CTA that opens the URL in a new tab.
