# Valuation model

This describes how ValYoued produces a structured valuation in code today. The authoritative implementation is `artifacts/api-server/src/lib/estimate.ts` (prompt assembly, news fetch, LLM call, math, fallbacks) plus `artifacts/api-server/src/lib/comparables.ts` (post-processing of comparable rows).

## Outputs the model must return

The LLM is asked for strict JSON (no markdown) that includes:

| Block | Role |
| --- | --- |
| `baselineLow` / `baselineMid` / `baselineHigh` | Core resale band in the seller currency |
| `comparables` | 3 to 5 comps with optional permalinks and match metadata |
| `marketSignals` | 3 to 5 labeled factors with multiplicative `impact` (about 0.85 to 1.20) |
| `worldEvents` | 3 to 6 news-linked events (see live headlines rules below) |
| `arbitrage` | Regional marketplace rows with fees, shipping, duties, `netToSeller` |
| `report` | Narratives: headline, summary, baseline/market/arbitrage/world-events/final |
| `proInsights` | Negotiation tactics, listing tips, walk-away and anchor prices (gated for some users) |

After the LLM responds, the server:

1. Parses JSON (with a fence-stripping helper).
2. Computes `adjustedLow` / `adjustedMid` / `adjustedHigh` by applying the average of `marketSignals.impact` as a net multiplier on the baseline band.
3. Normalizes `comparables` through `sanitizeComparables` (URLs, years, ordering). See [Prior listings and comparables](./PRIOR_LISTINGS_AND_COMPARABLES.md).
4. Clamps arbitrage when the asset is **not** internationally tradeable (single local row).
5. For **free** tier and tradeable assets, collapses arbitrage to one local or primary row (no full international grid in the product output).
6. Drops `proInsights` unless the subscriber is Pro **and** seller playbook is enabled for that request.

## Live news as a hard constraint for "world events"

Before the LLM runs, the server gathers articles via Google News RSS (`buildNewsQueries` + `searchNews` in `artifacts/api-server/src/lib/news.ts`). The prompt tells the model that `worldEvents` must cite those headlines (verbatim `source`, `url`, `publishedAt` where applicable) and explain price impact for **this** asset. If nothing fits, it may add at most one synthetic "General market context" row. This is **not** a separate econometric model; it is retrieval plus instruction-following.

## Category framing in the prompt ("everyday" vs "luxury / collectible")

The prompt encodes business rules that depend on asset class and seller notes:

- Mass-market categories (cars, electronics, furniture, ordinary home goods) should track recent comps and treat original purchase price as context unless evidence supports going above it.
- Luxury and collectible framing (watches, bags, vintages, sneakers, art, and similar) anchors to realized sales and credible asks rather than a generic label.

**Purchase price guardrail:** `adjustedMid` may exceed `purchasePrice` only when at least two comparables are recent sales above purchase and at least one `marketSignals` row has impact above 1.0 with a concrete reason. Otherwise the model is told to keep adjusted mid at or below purchase. This is enforced in prompt text, not as a separate numerical solver over the returned JSON.

## International arbitrage semantics

`AssetType.internationallyTradeable` (from `artifacts/api-server/src/lib/assetTypes.ts`) switches the arbitrage instructions:

- **Tradeable:** the model must produce several international rows with realistic shipping, fees, import friction, and exactly one `recommended: true` (highest net to seller).
- **Not tradeable (for example real estate, many cars):** the model must return a **single** local row only. The server also overwrites any stray multi-row output for these types.

## Fallback when the LLM fails

`fallbackCore` builds a heuristic band from condition and optional purchase price, fabricates placeholder comparables without URLs, neutral market signals, and minimal arbitrage. Logs record the failure path. Operators should treat fallback responses as degraded.

## Tiers and usage (business layer, not the neural "model")

Monthly free valuation caps and plan features are resolved in `artifacts/api-server/src/lib/entitlements.ts` before `generateEstimate` runs. The **`tier`** argument (`free` | `pro`) changes whether full arbitrage rows and `proInsights` appear in persisted results.
